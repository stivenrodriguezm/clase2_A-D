Todas las personas, colaboradores, stakeholders, que participan en un proceso de ingeniería de software. Gestion de negicio, gestión de proyecto, equipo de desarrollo, clientes y usuarios finales.

Es el SW como tal, y puede ir acompañado de: Documentación del proyecto, código fuente, documentos de prueba, documentos de cliente (manuales de usuario, de instalación, etc.)

Todo lo que involucran las fases para desarrollar el SW y tiene las siguientes fases: Planeación, Análisis de requerimientos, Diseño, Implementación, Pruebas, Mantenimiento.

Modelo de proceso de SW es la manera de organizar de una forma sistemática las personas y actividades para el desarrollo de un producto de SW en el marco de un proyecto de SW. Un ejemplo es el modelo de SW en cascada:
  1. Analisis
  2. Diseño
  3. Codificacion
  4. Pruebas
  5. Verificacion
  6. Mantenimiento
