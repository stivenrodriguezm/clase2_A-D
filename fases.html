Fase Actividades
Iniciación
  • Nace la idea del producto de SW
  • Identificación del cliente y segmentos del mercado
  • Entrevista a cliente/s para identificar necesidades
  • Análisis para determinar si existen productos similares
  • Se analiza la factibilidad del producto y posible aceptación
Planeación
  • Determinar recursos necesarios (Tiempo, cronograma, personal, etc.)
  • Determinar costo y tiempo de desarrollo
  • Ajustar el cronograma
  • Cambios de acuerdo con las necesidades del proyecto
  • Plan de gestión de la configuración: control de cambios y menejo de versiones de los diseños
Análisis
  • Recopilación de la informacion sobre necesidades de los clientes
  • Recopilación de informacion sobre funcionalidades del producto, seguridad, confiabilidad, usabilidad...
  • Recopilar requerimientos generales por medio de entrevistas, lluvia de ideas...
  • Los requerimientos definen que ees lo que el producto de software debe hacer
Diseño
  • Definir la arquitectura del SW y se diseña de forma especifica cada componente del software. todo se acompaña de la respectiva documentacion.
  • Hasta ahora no se programa nada
Implementación
  • Se programa todo el SW
  • El objetivo es tomar el diseño y progrmarlo completamente usando las tecnologias definidas
Pruebas
  • El SW se prueba para determinar su cumple con los requerimientos
  • 3 fases de pruebas:
  • - Se prueba cada modulo de forma independiente
  • - Se integran modulos para ver si la comunicacion es correcta
  • - Se determina si cumple con los requerimientos
  • Luego se implementan pruebas con los clientes y se pasa a 'Beta testing'
Mantenimiento
  • Comienza cuando el SW se libera completamente al cliente
  • Reparar fallos que el cliente encuentra
  • Implementar requerimientos adicionales
  • Se mejoran atributos como confiabilidad, desempeño, etc.
  • Después de cada mantenimiento se genera una nueva versión del SW
